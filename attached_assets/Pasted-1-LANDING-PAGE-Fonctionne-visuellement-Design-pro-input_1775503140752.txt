2. LOGIN PAGE — Le formulaire fonctionne, auto-fill Chrome OK. Le login API retourne 200. MAIS le cookie de session ne persiste pas entre les pages — c’est le BUG CRITIQUE #1.
3. SESSION / AUTH — BUG CRITIQUE #1
Le fichier server/auth/session-manager.ts ligne 157 override sameSite de "none" vers "strict" après chaque login. Avec SameSite=Strict, le cookie n’est PAS envoyé lors des redirections. Résultat: login → 200, puis la page suivante → 401 “Not authenticated”. Il y a aussi 2 configurations session conflictuelles (index.ts et routes.ts) avec des valeurs différentes pour secure, sameSite et maxAge.
Fix: Dans session-manager.ts, changer ligne 157 de session.cookie.sameSite = 'strict' → session.cookie.sameSite = 'lax'.
