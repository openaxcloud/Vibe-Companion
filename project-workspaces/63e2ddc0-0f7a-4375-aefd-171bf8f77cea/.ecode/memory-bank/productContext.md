# Product Context

## Problem Statement
Restaurant relies heavily on phone reservations, creating staff bottleneck during peak hours. Customers want to browse menu and make reservations online, especially post-pandemic.

## Target Users
**Primary**: Diners (25-55) seeking convenient reservation booking
**Secondary**: Restaurant staff managing bookings and menu updates
**Tertiary**: Food enthusiasts following chef's culinary journey

## UX Goals
- Seamless 3-click reservation process
- Visual menu browsing with dietary preferences
- Engaging blog content discovery
- Mobile-first experience for on-the-go users

## Key User Flows
1. Browse menu → Filter by dietary needs → View item details
2. Check availability → Select time/party size → Confirm reservation
3. Read blog posts → Subscribe to updates → Share content
4. Admin: Update menu items → Manage reservations → Publish blog posts
