# Project Brief – Memory Bank: E‑commerce Marketplace

1. **Project Name:** Memory Bank – AI-assisted context system for a TypeScript React full-stack e‑commerce marketplace.
2. **Purpose of Memory Bank:** Persist and surface important project context so the assistant can give consistent, high-quality, implementation-aware guidance over time.
3. **Product Summary:** A full-stack e‑commerce marketplace with:
   - Product catalog with search and filters
   - Shopping cart and checkout flow
   - Stripe payments
   - User authentication
   - Order management dashboard
   - Inventory tracking
   - Email notifications for orders
   - Mobile-responsive UI with dark mode
4. **Memory Bank Scope:**
   - Capture stable decisions (architecture, tech stack, conventions, domain model).
   - Record evolving domain concepts (product, order, inventory, user roles).
   - Track implementation state (what’s built, in-progress, blocked, deferred).
   - Remember recurring preferences (style, libraries, patterns).
5. **Non-goals:**
   - Not a full requirements spec or ticketing system.
   - Not a replacement for source control or formal docs.
6. **Core Requirements for Memory Bank:**
   - Be queryable by feature (e.g., “cart”, “payments”, “inventory”).
   - Keep a succinct, up-to-date view of the current architecture.
   - Capture API contracts and important schemas as they stabilize.
   - Track integration details with Stripe and email provider.
7. **Success Criteria:**
   - New conversations can quickly recover critical context.
   - Architectural consistency across new features.
   - Reduced repetition of previously-set decisions and patterns.
