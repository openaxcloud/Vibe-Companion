# Project Brief – Memory Bank for E‑commerce Marketplace

1. **Project Name:** Memory Bank – E‑commerce Marketplace (Full‑stack, TS + React)
2. **Objective:** Provide a structured, queryable long‑term memory for this AI to support consistent, context‑aware assistance across all phases of building the marketplace.
3. **Scope of Memory:**
   - Architecture decisions (frontend, backend, data, integrations).
   - Domain concepts (products, carts, orders, payments, users, marketplace vendors).
   - Implementation conventions (folder structure, coding standards, React/TS patterns).
   - API contracts, data models, and key flows (auth, search, checkout, Stripe, order management).
   - Open questions, trade‑offs, and resolved decisions over time.
4. **Out of Scope:**
   - Storing secrets (API keys, passwords).
   - User‑specific PII beyond abstract roles (buyer, seller, admin).
5. **Primary Goals:**
   - Preserve context across sessions (why decisions were made, not just what).
   - Speed up future answers with ready architectural and domain knowledge.
   - Maintain a single source of truth for requirements and constraints.
6. **Core Project Requirements to Capture:**
   - Full‑stack TS + React marketplace.
   - Product catalog with search & filters.
   - Shopping cart and multi‑step checkout.
   - Stripe payments integration.
   - User authentication and basic account management.
   - Order management dashboard (buyers, sellers, admins where applicable).
7. **Memory Bank Usage:**
   - As a reference for consistent terminology.
   - To avoid re‑asking previously decided design questions.
   - To guide incremental feature planning and refactors.
8. **Lifecycle:**
   - Updated iteratively as architecture, APIs, and UX flows evolve.
   - Older decisions annotated if superseded, not deleted.
