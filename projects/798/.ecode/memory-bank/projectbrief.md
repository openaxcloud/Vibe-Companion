# Project Brief – E‑commerce Marketplace Memory Bank

## Overview
- Memory Bank is a structured knowledge base for this e‑commerce marketplace project.
- It captures decisions, constraints, and patterns so the system remains coherent as it evolves.

## Core Requirements to Remember
- Full-stack TypeScript + React marketplace (multi-vendor capable in future).
- Product catalog with search, category filters, price range, and sort.
- Shopping cart with authenticated checkout flow (guest checkout optional later).
- Stripe integration for secure payments (Stripe Checkout or Payment Intents).
- User authentication (buyers + admin; vendors as future extension).
- Order management dashboard for admins (and later vendors) with status updates.

## Goals of the Memory Bank
- Preserve architectural decisions (APIs, data models, integrations).
- Track UX flows (search → cart → checkout → orders).
- Keep Stripe/payment constraints and security practices explicit.
- Support consistent naming, patterns, and tech choices across the codebase.

## Scope of This Document Set
- Covers: product & UX context, architecture patterns, tech setup, and immediate tasks.
- Excludes: marketing, content strategy, or non-technical business docs.
- Will be updated incrementally as decisions are made and revised.
