# Product & UX Context (Memory Bank)

## Problem Statement
- Teams need a focused, low‑friction messaging tool similar to Slack, without Slack’s complexity and cost.
- They require immediate communication, channel organization, and quick file sharing, all in the browser.

## Target Users
- Small to mid‑size software/product teams.
- Startup founders and cross‑functional squads needing quick coordination.
- Internal teams in larger orgs piloting a lightweight messaging solution.

## UX Goals
- Familiar Slack‑like mental model: workspaces → channels → messages.
- Real‑time feedback: typing indicators, instant message delivery, visible presence.
- Minimal clicks to switch context (channels/DMs) and share files.
- Responsive design optimized for desktop first, mobile‑friendly later.

## Key User Flows
- Auth: Sign up → create/join workspace → login → land in default channel.
- Channels: Browse/join public channels → create private channel → invite members.
- DMs: Start DM from user list or via search → send/receive messages in real time.
- Messaging: Type message → see typing indicators from others → messages appear in timeline instantly.
- Files: Attach file in composer → upload → file appears inline with preview/download link.

## Memory Bank Relevance
- Store these personas, flows, and UX goals to influence future design tradeoffs (e.g., prioritizing channel UX vs. admin features).
