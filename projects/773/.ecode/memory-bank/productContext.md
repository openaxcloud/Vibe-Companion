# Product & UX Context – Memory Bank

1. **Problem Statement:** Small merchants need a modern, secure, and easy‑to‑manage online storefront without custom‑coding everything from scratch.
2. **Target Users:**
   - Shoppers: browse, compare, and buy products on mobile/desktop.
   - Store Admin: manage products, inventory, and orders.
3. **UX Goals:**
   - Frictionless browsing and checkout; minimal clicks from discovery to purchase.
   - Clear feedback on stock, prices, discounts, and order status.
   - Accessible, mobile‑first design with a comfortable dark mode.
4. **Key Shopper Flows:**
   - Browse → search/filter → product detail → add to cart → checkout → Stripe payment → confirmation.
   - Auth: sign up / log in → manage addresses → view past orders.
5. **Key Admin Flows:**
   - Log in → view dashboard → filter orders → update order status (e.g., Processing → Shipped).
   - Manage catalog: create/edit products, images, prices, categories; adjust inventory.
6. **Usability Priorities:**
   - Fast perceived performance: skeleton loaders, optimistic UI for cart.
   - Trust: clear pricing, secure payment prompts, visible contact/support info.
   - Consistency: shared design system for forms, buttons, and alerts.
7. **Measurement:**
   - Funnel drop‑off (cart → payment), checkout completion rate, support tickets.
