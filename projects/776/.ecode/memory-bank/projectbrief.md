# Project Brief: Memory Bank for E‑commerce Marketplace

1. **Project Name:** Memory Bank – Context System for E‑commerce Marketplace Assistant
2. **Primary Goal:** Persist and surface the most relevant information (requirements, decisions, constraints, progress) to guide an AI assistant while designing and implementing the marketplace.
3. **Scope of Use:**
   - Support the build of a **full‑stack TypeScript/React e‑commerce marketplace**.
   - Features: Stripe payments, product catalog with search/filters, shopping cart + checkout, user auth, order management dashboard, inventory tracking, order emails, responsive UI with dark mode.
4. **Memory Bank Responsibilities:**
   - Capture **stable project facts** (requirements, tech stack, non‑functional constraints).
   - Track **architectural decisions** and patterns.
   - Record **active tasks, TODOs, and open questions**.
   - Provide **quick recall** to keep the assistant consistent over multiple interactions.
5. **Out of Scope:**
   - Storing large code blobs or logs.
   - Acting as a task runner or CI/CD system.
6. **Key Objectives:**
   - Reduce repetition of requirements.
   - Avoid architectural drift or contradictions.
   - Speed up onboarding for anyone querying the assistant.
7. **Success Criteria:**
   - The assistant answers stay aligned with this marketplace’s goals.
   - New design or implementation requests respect earlier decisions.
   - Project phases (setup → MVP → polish) are clearly tracked in memory.
