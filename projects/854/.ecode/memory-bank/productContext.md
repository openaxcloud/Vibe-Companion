# Product Context

## Problem Statement
Create a Slack-like real-time messaging platform with WebSocket connections, public and private channels, direct messages, file sharing, typing indicators

## Target Users
- Primary users who need this functionality
- Secondary users who may benefit

## User Experience Goals
- Intuitive interface
- Fast performance
- Accessible design

## How It Should Work
1. User accesses the application
2. Core functionality is immediately available
3. Data is persisted across sessions
