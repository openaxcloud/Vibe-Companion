# Product & UX Context – Memory Bank

## Problem Statement
- Teams need a lightweight, Slack-like tool for organized real-time communication without the complexity of enterprise suites.
- Existing tools are either too heavy or lack fine-grained channel control and modern UX.

## Target Users
- Small to mid-size software teams collaborating daily.
- Startup founders and project squads needing dedicated channels and DMs.
- Internal departments (e.g., support, ops) requiring real-time coordination.

## UX Goals
- Familiar, low-friction UI: channel list, message pane, and details sidebar.
- Fast, responsive input with instant message echo and optimistic UI updates.
- Clear separation of workspaces, channels, and DMs.
- Safe and intuitive file sharing: drag-and-drop, previews, and clear size limits.

## Key User Flows
- Authentication: sign up, sign in, password reset; join or create workspace.
- Channel usage: browse/create public channels, invite users to private channels, join/leave.
- Messaging: send/receive messages in real-time, edit/delete own messages (later phase), view timestamps and read status (basic in future phase).
- Direct messages: start 1:1 or small group DM from user list or message context.
- File sharing: attach files to messages, upload, and access from message history.
- Typing indicators: show when other participants in the current channel/DM are typing.

## Memory Bank Usage
- Capture evolving UX decisions (e.g., input behavior, shortcuts, layout changes).
- Track user feedback and translate into prioritized UX improvements.
