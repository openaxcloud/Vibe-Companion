# Project Brief – Memory Bank: E‑Commerce Marketplace

## Overview
- Memory Bank is the structured knowledge base for designing and building a full‑stack TypeScript/React e‑commerce marketplace.
- It captures persistent context: goals, constraints, decisions, and patterns the team will reuse over time.

## Core Product Requirements
- Marketplace web app with product catalog (browse, search, filter, sort).
- Shopping cart and multi-step checkout with Stripe payments.
- User authentication (sign up, login, sessions, password reset).
- Order management dashboard (for customers and admins/merchants).

## Memory Bank Goals
- Centralize project knowledge so decisions are consistent and discoverable.
- Provide a shared language for product flows (cart, checkout, orders).
- Record architecture and API contracts for front-end/back-end integration.
- Track trade-offs (e.g., Stripe integration patterns, data modeling).

## Scope of This Documentation
- Define what the system does and for whom (product context).
- Capture high-level architecture and recurring design patterns.
- Describe tech stack, tooling, and environment assumptions.
- Maintain an “active context” section to guide near-term work.
