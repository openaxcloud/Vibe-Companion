# Project Brief – Memory Bank for E‑commerce Marketplace

1. **Project Name:** Memory Bank – E‑commerce Marketplace Assistant
2. **Objective:** Persist and organize long‑lived project knowledge (decisions, constraints, patterns) to guide consistent evolution of a TypeScript/React full‑stack marketplace with Stripe payments.
3. **Primary Function:** Act as a structured memory layer for all aspects of the marketplace: product catalog, search/filters, cart/checkout, auth, payment, and order management.
4. **Scope of Memory:**
   - Business rules (commission, fees, refund policy, marketplace roles).
   - UX flows (search, cart, checkout, dashboard behaviors).
   - Technical decisions (API shape, data models, integrations, security choices).
   - Operational context (environments, secrets, deployment, observability).
5. **Out of Scope:** Source code storage, secrets, and transient logs.
6. **Core Requirements:**
   - Capture decisions and rationale as the system evolves.
   - Provide quick recall for recurring questions (e.g., discount rules, search behavior).
   - Maintain consistency across backend, frontend, and infrastructure.
7. **Goals:**
   - Reduce re‑decision and contradictory implementations.
   - Support onboarding and knowledge transfer.
   - Enable traceable evolution of marketplace features and policies.
8. **Consumption:** Read and write by developers, PMs, and AI assistants via structured notes, tags, and links.
9. **Lifecycle:** Continuously updated; older entries versioned instead of deleted.
10. **Success Criteria:** Fewer conflicting requirements, faster feature design, and predictable behavior of Stripe payments, auth, and order flows.
