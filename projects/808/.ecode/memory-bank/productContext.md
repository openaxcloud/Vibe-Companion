# Product & UX Context

## Problem Statement
- Teams need a lightweight, focused real-time chat tool for fast coordination.
- Existing tools can be heavy, noisy, and complex for small/medium teams.
- This product aims to provide essential Slack-like functionality with simpler UX.

## Target Users
- Small to mid-sized teams (5–200 people).
- Engineering/product teams needing structured channels and private groups.
- Cross-functional teams coordinating projects in real time.

## UX Goals
- Make channel and DM navigation frictionless and predictable.
- Keep message composition fast (keyboard-friendly, minimal clicks).
- Provide clear feedback on message delivery and connection status.
- Ensure users immediately see which channels are active and relevant.

## Key User Flows
1. **Sign in & Onboarding**
   - Log in → see workspace overview → auto-join default public channels.
2. **Channel Management**
   - Create public/private channel → invite members → manage membership.
3. **Messaging**
   - Select channel/DM → view history → send messages/files → see typing indicators.
4. **Direct Messages**
   - Search/select user → open DM → exchange real-time messages and files.
5. **Awareness & Presence**
   - View who is online → see typing in current room → see new message indicators.
