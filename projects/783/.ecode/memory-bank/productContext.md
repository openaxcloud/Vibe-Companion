# Product & Memory Context

## Problem Statement
- Teams need a focused, real-time messaging space where information does not disappear but can be recalled contextually.
- Existing tools often bury knowledge in long message streams without structured memory or summaries.

## Target Users
- Small to mid-sized software/knowledge teams (5–200 people).
- Heavy async communicators who rely on channels and DMs for day-to-day work.

## UX Goals
- Familiar Slack-like experience to reduce onboarding friction.
- Fast navigation between channels/DMs with unread badges and mention highlights.
- Seamless file sharing with inline previews and links.
- Memory Bank elements visible but unobtrusive: summaries, pinned key decisions, and quick access to past context.

## Key User Flows
1. **Sign in & join workspace** → see channel list → auto-join default channels.
2. **Browse channels** → open a public channel → read history → send messages → see real-time updates.
3. **Create private channel** → invite members → share files → view stored summary / pinned items.
4. **Start a DM** → select user(s) → send messages → see typing indicators and last-seen.
5. **Search memory** → query across channels/DMs → filter by user/channel/date → open result in context.
6. **Memory curation (future)** → mark a message as “Important” → add to Memory Bank for that channel.
