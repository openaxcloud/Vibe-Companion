# Memory Bank – Project Brief

## Overview
- Memory Bank is an internal knowledge system that tracks key decisions, context, and rationale for the e-commerce marketplace project.
- It complements formal docs by capturing evolving insights, trade-offs, TODOs, and lessons learned.

## Purpose in This Project
- Preserve architectural and product decisions about the full-stack TypeScript/React e-commerce marketplace.
- Provide quick onboarding context for new contributors (product flows, tech stack, constraints).
- Reduce rework by recording why choices were made (e.g., Stripe integration mode, search strategy, auth design).

## Scope
- Covers all layers: product, UX, architecture, infra, security, and operations.
- Tracks decisions around: Stripe payments, product catalog & search, filters, cart & checkout, auth, order dashboard.
- Stores implementation notes: folder structure, key libraries, integration approaches, testing strategy.

## Constraints & Principles
- Lightweight: fast to read, fast to update; no heavy process.
- Truth-oriented: favor clarity over completeness; record disagreements & alternatives.
- Versioned with code: Memory entries evolve alongside the repository.
- Organized by topic: Product, System, Tech, and Active Context files as primary indexes.
