# Project Brief: Simple Todo App with Task List

## Overview
- Build a minimal but robust web-based todo application that lets users create, view, update, and delete tasks.
- Focus on clarity, responsiveness, and maintainable architecture suitable for incremental enhancements.

## Core Requirements
- Add new tasks with a title (required) and optional description.
- Display tasks in a list with clear status (e.g., pending / completed).
- Toggle task completion state from the list.
- Edit and delete existing tasks.
- Persist tasks locally in the browser so they survive page refreshes (e.g., localStorage).
- Basic filtering: view all / active / completed tasks.

## Goals
- Provide a clean UX that makes capturing and managing tasks fast and frictionless.
- Use a simple, testable React + TypeScript architecture.
- Prepare the codebase for future enhancements (e.g., due dates, tags, multi-lists).

## Scope
- Single-page React app, no backend service.
- Desktop-first layout with usable mobile view.
- No authentication, user accounts, or collaboration features.
- No external state management library (use React state + custom hooks).
- Memory Bank documents decisions, patterns, and next steps for ongoing development.
