# Memory Bank — Project Brief

- **Project Name:** Full-Stack E‑Commerce Marketplace
- **Memory Bank Purpose:** Persist and surface key project decisions, constraints, and context so the team stays aligned over time.
- **Core Product Goals:**
  - Marketplace for multiple sellers to list products.
  - Product catalog with search, filters, and sort.
  - Shopping cart and guided checkout flow.
  - Stripe-based payments and refunds.
  - User authentication (buyers, sellers, admins).
  - Order management dashboard (status, fulfillment, history).
- **Scope (MVP):**
  - Web application (TypeScript + React front end, matching TypeScript backend).
  - Basic marketplace model: users, products, orders, carts, payments.
  - Essential dashboards and reports; no mobile app.
- **Out of Scope (MVP):**
  - Advanced recommendation engine, marketplace disputes, multi-currency, complex tax automation.
- **Memory Bank Usage:**
  - Capture evolving decisions (APIs, data model, auth, Stripe usage).
  - Record constraints (performance targets, compliance, team preferences).
  - Maintain active tasks and context for handoffs.
- **Success Indicators:**
  - Team can onboard in <1 day using Memory Bank docs.
  - Architectural and product decisions are discoverable and not re-litigated.
  - Shipping MVP without major rework due to forgotten decisions.
